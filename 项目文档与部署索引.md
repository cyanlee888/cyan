## 4. Demo 同步 SOP（改完 html 后）

```bash
SRC="/Users/lishuang/Documents/Cursor/Dino English"
DST="/Users/lishuang/Documents/Cursor/dino/cyan/docs/dino-english"

# 1) 复制 html（按需替换文件名，如 V1.3.0-ui-demo.html）

# 1b) 若改/加了图片，同步对应 assets，例如：

git add docs/dino-english/
git commit -m "Update dino-english demo"
git push origin main

# 3) 验证：线上与本地 hash 应一致（构建约 1–2 分钟）
curl -s https://cyanlee888.github.io/cyan/dino-english/V1.2.0-ui-demo.html | shasum
shasum "$SRC/V1.2.0-ui-demo.html"
```

**图片资源**：demo 用相对路径引用 `assets/...`，必须把 html **连同改动到的 assets** 一起复制到 `docs/dino-english/`，否则线上破图 / 卡片变形。

## 5. 缓存说明（线上没变 ≠ 没部署）

- GitHub Pages `cache-control: max-age=600` → CDN/浏览器最多缓存 **10 分钟**。
- push 后构建约 1–2 分钟，期间线上还是旧版属正常。
- 看到旧版/“变形”先排除缓存：**硬刷新 `Cmd+Shift+R`**，或链接加 `?v=2` 强制绕过；用上面的 hash 比对确认是否真未同步。
