## 前言

V1.3.0 在 V1.2.0 商业化闭环之上推进两条主线：

- **游戏化激励体系**：Dino 成长（XP / 等级）、恐龙币经济、商城 / 背包、课后 Play 游戏（Dino 盲盒机 / 单词 PK / 口语 PK）与课中学习奖励，配套会员升级引导。
- **功能性迭代**：定级测试（嵌入 Onboarding Step4，1 道英语水平自评单题 → 推荐 CEFR 档位）；体验课学习报告（竖屏，老师点评 + 定级 + 学习情况 + 课堂所学 + 发音分析 + 家长 CTA）；Dino Tab 对话重设计（打招呼气泡 + 大语音按钮 + 沉浸式对话页）；Explore 新架构（Words & Sentences / Listening / Speaking 分类入口 + 二级内容页）；解锁 & 学习计划（自然 7 天滚动解锁 3 节 · 按进度锚定 · 不按周几 + Learning Path 页）；Paywall 商品为周付 + 月付 + 年付三档（年付默认选中、首推）。

应用导航升级为底部 4 Tab（Dino / Class / Explore / Play，默认 Class）+ 左上角头像进入个人中心；全应用界面文案为英文；主流程与免费 / 付费权益边界沿用 V1.2.0。

## 四、需求背景

### 产品愿景与定位

**愿景**：让孩子每天愿意主动开口，与 AI 老师在沉浸式教学故事里把英语说出来。

**定位**：对于 6-13 岁、缺少英语交流环境的儿童家庭，Dino English 是一个 AI 英语开口陪练产品，帮助孩子在互动中敢说、常说、说对。不同于传统录播课和真人外教课，以 AI 老师贯穿叙事与教学，游戏化机制为玩法和驱动承载，让孩子低压力、高频次地练习表达。

> 免费用户超出当日游戏次数时提示并引导升级会员（trigger_source=game_limit），不强制付费；盲盒券（开盒凭证）与课程权益边界不受限次影响。会员「不限次玩」是核心付费卖点（随时可玩），每次完成仍给恐龙币，但从第 2 次起递减、第 6 次起仅保底 +5，避免“学了但没奖励”的断崖感。

### 重复使用奖励规则（首次 vs 复习）与上限

> 核心原则：**奖励跟「学习进度」而非「重复次数」走**。学习内容首次完成发全额，重复使用已通关内容只发「复习奖励」以防刷；游戏与 FM 因本身可重复、随机或被动消费，靠「当日同类次数递减 + +5 兜底」控制，无需区分首次 / 重复。

> 数值不在此重复，统一见上「XP 与恐龙币发放总表」；本表只列复习 / 重复口径与防刷机制。

| 内容类型 | 复习 / 重复使用规则与防刷机制 |
| --- | --- |
| 主课程 | 重复学已通关：**仅发减半 XP（+25）、不发恐龙币**；按 lesson_id「是否已通关」判定，计入每日复习 XP 上限（建议 +75 XP/天，约 3 节复习封顶），复习不重复产币 |
| Words & Sentences | 重复学已通关：**仅发减半 XP（+15）、不发恐龙币**；按 story_id 判定，并入每日复习 XP 上限统一计 |
| Listening（FM） | **不区分首次 / 重复**，按当日 Listening 条数递减；第 6 条起每条仍发 +5 恐龙币 |
| Dino 盲盒机 | **不区分首次 / 重复**（题目随机、本就可重复）；免费每日 1 轮、会员不限轮玩，按当日轮次递减；第 6 轮起每轮仍发 +5 恐龙币 |
| 单词 PK / 口语 PK | **不区分首次 / 重复**（对局随机、本就可重复）；免费每日各 1 局、会员不限局玩，按各自当日局数递减；第 6 局起每局仍发 +5 恐龙币 |

要点：

- **恐龙币只在「首次完成学习内容」与「游戏 / FM 递减奖励」中产出**；重复学已通关的主课 / 练习**不再产币**，堵住「反复刷同一节课刷币」的唯一漏洞。
- **复习仍给少量 XP**（减半），保留「多练有正反馈」的鼓励，但用复习 XP 子上限（+75/天）与全局每日 XP 上限（默认 500/天）防止刷等级。
- **FM 与游戏不做首次 / 重复区分**：免费游戏靠每日次数控制，会员与 Listening 靠递减 + +5 兜底控制，保证每次完成都有奖励但高频收益明显变低。
- 是否「已通关 / 已学过」以内容完成记录（主课 lesson_id、W&S story_id）判定；复习奖励数值与每日复习上限均由运营后台配置。

### 当日多内容奖励递减与每日封顶（防「一天刷多内容」）

> 除「重复刷同一内容」外，对「一天内完成多个内容」也做温和的**恐龙币递减**，避免单日刷币；**学习主线（主课）不递减**，鼓励多上课。

- **适用对象（按次递减仅作用于恐龙币；XP 不按次递减、改用每日上限，见下）**：Words & Sentences、Listening、三款游戏（Dino 盲盒机 / 单词 PK / 口语 PK）。**主课不递减**——主课首次完成照常全额，重复学已通关主课仍按「减半 XP、不发币」。
- **递减口径（按「类别」当日完成次数，各类别独立计数、次日 0 点重置）**：`100% / 80% / 60% / 40% / 25% / +5` 是**当次恐龙币相对该行为基准值的折算比例与兜底值**——当日第 1 次发 100%（全额）、第 2 次 80%、第 3 次 60%、第 4 次 40%、第 5 次 25%、**第 6 次起每次固定 +5 恐龙币**；比例折算后四舍五入取整，低于 5 时按 5 发放。
- **XP 不按次递减、改用「全局每日上限」控速**：每次行为的 XP 仍按基准**全额**发放（不随当日次数递减——恐龙币用「按次递减」控经济、XP 用「每日上限」控升级节奏，各用更适配的工具，体感上避免「做同样的事却几乎不长经验」的惩罚感）；新增**全局每日 XP 上限**：**默认 500 XP/天（运营可配）**，当日累计 XP 达上限后，后续任意行为统一只发 **+5 XP 兜底**（保留正反馈、不归零），次日 0 点重置。**主课首次完成的 XP 不计入该上限、始终全额**（主课受「7 天滚动解锁」天然限量，鼓励多上课）；Words & Sentences / Listening / 三款游戏 / 以及所有「复习」产生的 XP 均计入。
- **复习 XP 子上限（含在全局上限内）**：重复学已通关主课 / W&S 仍仅发减半 XP（+25 / +15）、不发币，且复习 XP 单设子上限 **+75 XP/天**（约 3 节复习封顶），防止当日 XP 全靠重复刷已通关内容堆出。
- **各类别示例**：Words & Sentences 基准 40 → 40 / 32 / 24 / 16 / 10 / 5…；Listening 单条 12 → 12 / 10 / 7 / 5 / 5 / 5…；Dino 盲盒机每轮 30 → 30 / 24 / 18 / 12 / 8 / 5…；PK 胜场 40 → 40 / 32 / 24 / 16 / 10 / 5…；PK 平 / 负 20 → 20 / 16 / 12 / 8 / 5 / 5…；主课不在此列（永不递减、始终全额）。
- **不设“第 5 次后停发”硬上限**：高频行为继续给 +5 兜底，保证每次学习 / 完成都有正反馈；经济收紧通过递减曲线与皮肤涨价实现。
- **全局每日恐龙币上限：本期不设**（移入需求池）——「免费游戏每日次数」＋「会员 / Listening 当日同类递减」已能控制单用户日获币速度，无需叠加全局硬上限。
- **示例（恐龙币）**：当日玩 8 轮盲盒（每轮基准 30）→ 30 / 24 / 18 / 12 / 8 / 5 / 5 / 5 = 107 币；当日完成 3 篇不同 W&S（每篇 40）→ 40 / 32 / 24 = 96 币；当日连续听 8 条 Listening → 12 / 10 / 7 / 5 / 5 / 5 / 5 / 5 = 54 币；**XP 每次均按基准全额、不随次数变化**（但当日累计达 500 XP 上限后，后续每次仅 +5 兜底；主课首次完成的 XP 不计入上限）；主课无论当天上几节均全额发。
- 递减比例与 +5 兜底由**运营后台配置**。

**4）防刷分层（不靠单一「游戏获币上限」）**：
- **学习内容（主课 / W&S）**：内容驱动、天然有限；**重复学已通关内容不再发恐龙币**（仅发减半 XP，见「重复使用奖励规则」），堵住「反复刷同一节课刷币」的唯一漏洞。
- **Listening**：不再 5 条后停发，改为按条数递减并 +5 兜底；高频收听有奖励但收益很低。
- **免费用户游戏**：靠每日次数（Dino 盲盒机 1 轮 / 单词 PK 1 局 / 口语 PK 1 局）限制，无需额外币上限。
- **会员游戏**：兑现「不限次玩」承诺（随时可玩），每类玩法每次完成都发币，但从第 6 次起仅 +5；恐龙币纯外观（非 pay-to-win），递减只影响外观解锁速度、不影响学习与 PK 体验。
- **当日多内容递减**：W&S / Listening / 三款游戏按「类别当日完成次数」对恐龙币按 100/80/60/40/25/+5 递减（**主课不递减、XP 不受影响**），抑制「一天刷多内容」（见「当日多内容奖励递减与每日封顶」）。
- **全局每日恐龙币上限**：本期不设（移入需求池）——由「免费游戏每日次数 + 高频递减兜底 + 皮肤涨价」共同收紧经济。
- **全局每日 XP 上限**：默认 500 XP/天（运营可配），达标后后续行为统一 +5 兜底、主课首次完成不计入；用于控制升级节奏、防止刷等级（XP 不按次递减，详见「当日多内容奖励递减与每日封顶」）。

**5）结论与原则**：
- 付费墙建立在**学习内容（主课）**而非外观上：免费用户可通过游戏慢速攒币兑换皮肤（提升打开率与留存），但持续学习价值仍需会员，经济模型与商业模型一致、不冲突。
- 活跃会员「每天都有币入账、几天攒出进阶款、约一周攻克稀有款」，正反馈密度合理且高档形成长期目标（全部集齐约 ~40 活跃天）；非活跃用户攒币明显变慢，形成自然的活跃牵引而非硬付费门槛。
- XP 升级节奏：活跃会员日均约 160–180 XP（抽盒不再发 XP，仍在 500 每日上限内、正常不触顶），约 2 天到 Hatchling（300）、约 5 天到 Brave Dino（800），与攒币节奏同频，避免「等级 / 货币」两套进度互相拖累。

## 八、Play 游戏出题规则（与教研对齐用基础口径）

> 本章为产品侧基础规则，用于与教研团队对齐题库分层与出题口径；具体题目内容、词表与难度校准由教研侧产出。

### 总原则

- 三款游戏（Dino 盲盒机 / 单词 PK / 口语 PK）**统一按用户当前 Level（L1–L6，由定级测试推荐 + 课程进度）取题**，不做答题中实时自适应；答题页不展示难度 / 目标答对率等文案。
- 题库按 Level 分层维护：每题打 `level` 标签（L1–L6），与课程体系 6 个 Level 一一对应。
- **出题范围 = 用户当前 Level 及以下的全部题库（≤ 当前 Level）**：范围内均为已学 / 在学内容，不向上取更高 Level 的新题。
- **优先「未答过」的题**：后台维护每用户「已答题 ID 集合」，每轮**先在范围内随机抽取用户未答过的题**；未答过的题不足时，再从**已答过的题**中随机补足，自然形成复习回灌。
- 目标体感：单轮答对率约 70%–80%（由「≤ 当前 Level 范围 + 题库难度配比」实现，不做实时调整）；连续答错不降级、连续答对不升级，Level 仅随课程进度 / 定级更新。
- 同一轮内题目不重复；跨轮新鲜度由「已答题集合 + 未答优先」保证（已答过的题作复习随机回灌）。

### 各游戏题型与分层口径

| 游戏 | 题型（本期） | L1–L2（Pre-A1 / A1） | L3–L4（A1+ / A2） | L5–L6（A2+ / B1） |
| --- | --- | --- | --- | --- |
| Dino 盲盒机 | 四选一选择题（图 / 词 / 句匹配） | 看图选词、听音选图：单词级（高频名词 / 颜色 / 数字 / 动物） | 词义配对、补全短句：短语与简单句（日常场景、一般现在时） | 选句补全对话、近义辨析：复合句与情景表达（时态扩展、常用功能句） |
| 单词 PK | 限轮次对战 · 四选一词汇题 | 高频基础词（拼写 ≤ 6 字母为主），干扰项差异大 | 课程主题词 + 常用搭配，干扰项同类词 | 低频词 / 多义词 / 搭配辨析，干扰项近义词 |
| 口语 PK | 跟读 / 朗读对战（**本期接入发音评分模型，复用课中已有发音评测能力**） | 单词与短语跟读（2–4 词） | 整句朗读（5–8 词，含目标句型） | 短段落 / 角色扮演句（8 词以上，含连读语调） |

### 出题流程（产品口径）

1. 进入游戏时读取用户当前 Level（无定级结果时默认 L1）。
2. 构建候选池 = **当前 Level 及以下全部题库**；**优先随机抽取未答过的题**，未答过的不足再随机抽已答过的题（Dino 盲盒机 / 单词 PK 每轮 5 题，口语 PK 每轮 3 句），同一轮不重复。
3. 候选池整体不足时才向上借题，并标记回填需求给教研。
4. 答题数据（按题 `level` / 正确率 + 已答标记）回流教研，更新用户「已答题集合」并用于难度校准与题库迭代。

## 九、假 PK 对战专题（单词 PK / 口语 PK）

> 单词 PK 与口语 PK 本期均为**假 PK**：单机模拟对战，无真实对手、无实时联机匹配，胜负按预设胜率判定。本章定义匹配、胜负判定、答题与结算规则，参数均可由运营后台配置。

### 9.1 对手匹配池（Rival Pool）

- 进入 PK 后展示**匹配页**（约 2 秒）：己方头像固定，对手头像在对手池中轮播 +「Finding your rival…」+ 加载动效；匹配完成定格为一名对手（名 + 卡通头像）。匹配中点返回可退出。
- 对手池为**固定配置的 AI 形象**（本期 8 名，可运营扩充）：Leo / Mia / Max / Zoe / Sam / Lily / Ben / Ava；每次对局从池中随机取一名。
- 对手仅作展示与分数模拟，不关联任何真实用户数据。

### 9.2 胜率控制（核心规则）

胜负**在匹配完成时即按预设胜率一次性预先判定**（win / draw / loss），不依赖用户答题快慢；答题过程只决定「赢多少 / 输多少」的比分，不改变已判定的胜负方向（极端比分按边界回填，见 9.4）。

| 参数 | 含义 | 本期默认值 | 配置方 |
| --- | --- | --- | --- |
| 新手保护局数 `newbieMatches` | 前 N 局使用更高胜率 | 3 局 | 运营 |
| 新手胜率 `newbieWinRate` | 新手保护期内判定为「胜」的概率 | 90% | 运营 |
| 稳态目标胜率 `targetWinRate` | 新手保护期后判定为「胜」的概率 | 75% | 运营 |
| 平局占比 `drawShare` | 非胜空间中判为「平」的比例（其余为负） | 30% | 运营 |

- 判定逻辑：取当前胜率 `wr`（前 3 局 90%，之后 75%）→ 随机数 < `wr` 判**胜**；落在 `wr` 之上的剩余区间里，前 30% 判**平**、其余判**负**。
- 稳态期约：胜 75% / 平 7.5% / 负 17.5%（数值随配置变化）。
- 胜率按「已完成 PK 局数」推进新手保护窗口；该计数与每日限次相互独立（限次仅控制可玩次数）。

### 9.3 答题规则

- 单词 PK 与口语 PK **均不设逐题倒计时**（`perQuestionSec=0`）：作答 / 录音完成即进入下一题，不比拼答题速度；胜负控制见 9.2。
- 单词 PK：每轮固定 5 题、四选一，按用户当前 Level 取题（见第八章）；答题页展示题号 Question n / 5 + 实时比分（You vs 对手），不展示恐龙币余额；选项作答后即时反馈（对：高亮 +1；错：标红并回显正确项）并进入下一题。
- 口语 PK：每轮 3 句跟读 / 朗读，按用户当前 Level 取题；每句进入时先自动播放示范音频（句子 / 单词，可点「Play audio」重听）→ 跟读录音 → 接入发音评分模型（复用课中已有发音评测能力）对该句打分、按得分计入本局比分 → 点 Done 进入下一题；胜负仍按预设胜率判定，评分只决定比分高低、不改变预设胜负方向；其余匹配与胜率框架与单词 PK 一致。

### 9.4 对手分模拟（实时跟随，不回填）

- **核心改为「实时跟随」**：对手分不再独立随机、也不在结束时回填覆盖；而是每题作答后朝「用户当前真实分 ∓ 让分 `margin`」推进，**每题最多 +1、全程单调不回退**。因此**过程分 = 最终分**，结果页不会出现对手分突然跳变或缩水。
- **让分 `margin`**：开局随机取 **1 或 2**（约 40% 取 2），整局固定，决定最终领先 / 落后的分差。
- **方向跟随 9.2 预判结果**：
  - 判**胜**：对手贴在用户分之下，最终对手分 = 用户分 − `margin`（下限 0）。
  - 判**平**：对手分 = 用户分。
  - 判**负**：对手贴在用户分之上，最终对手分 = 用户分 + `margin`（上限为总题数）。
- **两个端点自然收敛为平**（脚本方向无法兑现时按真实 / 就近结果呈现，不强行改判）：
  - 用户**满分**却判负：对手最高只能到满分 → 呈现**平局**（满分 : 满分，体感为「最后一题被追平」）。
  - 用户**0 分**却判胜：对手最低只能到 0 → 呈现**平局**（0 : 0）。
- **真实低分胜保留**：用户 ≥ 1 分且判胜时，允许出现如 1 : 0 的真实低分胜；仅用户 0 分判胜降为平。判负 + 用户 0 分仍真实判负（如 0 : 1）。
- **胜率口径**：`win = 用户分 > 对手分`，平 / 负均不计 `wins`；实际胜率 = 9.2 脚本胜率 × P(用户至少答对 1 题)，仅「判胜但用户全错」会由胜降为平，其余仍由 9.2 严格控制，对总体胜率影响极小。

### 9.5 结算、胜负展示与限次

- 完成后**直接进入合并的对战结果页**：以**对战卡片**形式同屏展示「双方头像 + WIN / LOSE / TIE 标签 + 累计胜率（**不展示段位**）+ 最终比分（含对手名）」与「本局奖励：胜 +30 XP / +40 恐龙币；平 / 负 +15 XP / +20 恐龙币」，进入即发放并写流水（单词 PK `source=word_pk`、口语 PK `source=speaking_pk`，两线同口径）。
- 结果页主按钮 **Continue → 回 Play Tab**。
- **累计胜率 = wins / played**（按该游戏线分别统计），是对战实力的唯一外显；假 PK 的胜负同样计入。
- 免费用户每日各 1 局（单词 PK / 口语 PK 分别计）、会员不限玩且按当日同类局数递减，第 6 局起每局 +5 恐龙币兜底；超限提示并引导升级（trigger_source=game_limit），不强制付费。

### 9.6 对战胜率展示

- **展示位置**：Play 入口卡（`Win rate XX%`）、对战结果页（对战卡片下方 `Win rate XX%`）。首局前无数据时显示「—」。
- **计算口径**：`win rate = wins / played`（该游戏线累计；单词 PK 与口语 PK 各自独立统计、互不影响）。胜 / 平 / 负中，仅「胜」计入 `wins`，平、负不计 `wins` 但都计入 `played`。
- **不展示段位**：不展示段位名称 / 图标、段位升级提示、升级门槛（场次 / 胜率阈值）；底层 `wins` / `played` 持久化供数据分析，不向用户呈现任何段位层级。

## 十、学习计划页设计专题

> 覆盖「体验课报告 → 学习计划领取页 → Paywall」链路（**仅免费用户**；体验课 lesson0 首次完成后不可重上、报告入口保留可复看，会员态报告底部按钮直接 Continue 回 Class、不进领取页 / 不跳 Paywall）。领取页不展示周几 / 本周清单，主体为 Personalized learning plan 三张价值卡（12 units · 144 lessons / 3 lessons a week / Level X in 12 months）。

### 10.1 页面结构（三层）

| 层 | 内容 |
| --- | --- |
| 顶部 · 页面主标题 + CEFR 对齐卡片 | 页面主标题「{name}'s English Journey」；卡片区标题「Aligned with CEFR international standards」+ 两张卡——左卡 CURRENT（当前 Level + 对应 CEFR）、右卡 NEXT GOAL（当前 Level 设定的能力目标，按 Level 取，承接体验课报告定级）|
| 中部 · Personalized learning plan 价值卡 | 三张价值卡：① 12 units · 144 lessons（按当前 Level 计数，Unit / lesson 数按实际取数据库；副「Multiple lesson types tailored to each level: Vocabulary · Sentences · Reading · Phonics · Writing · Grammar · Review」）② 3 lessons a week（「3」为可配置变量，不绑定周几、不设休息日；副「Short lessons weekly. Steady progress ahead.」）③ Level X in 12 months（X = 下一个 Level、最高封顶 Level 6；副「A clear goal. Easy to track.」）|
| 底部 · The full 6-level system | Level 列表（6 Level 连线轨道，展示 Level 与对应 CEFR：L1 Pre-A1 / L2 A1 / L3 A1+ / L4 A2 / L5 A2+ / L6 B1）+ Unit 列表（当前 Level 的 Unit 1~12 教研主题连线时间轴）|

### 10.2 内容粒度

- 默认展示当前 Level 的 Unit 主题（12 站），不平铺 144 节（12 Unit × 12 Lesson）；每 Unit 12 节细节在 Level和unit列表页（Learning Path）查看。
- 数字口径：The full 6-level system 展示 6 个 Level 与对应 CEFR（Level / Unit 数按实际取数据库、不写死）；价值卡只展示当前 Level 的 Unit / lesson 数（12 units · 144 lessons），不写 6 个 Level 总量；口径为「已上线 + 持续更新」，不得把全量架构当现货承诺。

### 10.3 内容展示与 Level 闭环

- 不暴露空课位：近期解锁课真实展示；Level 远端、暂未生产的部分用 Unit 主题地图承载（主题确定可见，具体 Lesson 随产能逐步点亮）。
- Level 闭环：每个 Level 配 1 节体验课（该 Level 第 1 节，免费）。onboarding 自评模糊定级 → 进对应 Level 体验课 → 首节体验课报告产出正式定级 → 据此生成学习计划、确定起始当前 Level（学完当前 Level 后自然进入下一 Level，定级随之上移）；6 个 Level 各 1 节 = 6 节体验课。

### 10.4 Level → CEFR 映射与展示口径

- Level 标签统一加「CEFR」前缀，描述项需多语言：

| Level | 标签 |
| --- | --- |
| L1 | CEFR Pre-A1 · 零基础 |
| L2 | CEFR A1 · 单词积累中 |
| L3 | CEFR A1+ · 短句读与说 |
| L4 | CEFR A2 · 短文读写 |
| L5 | CEFR A2+ · 独立表达 |
| L6 | CEFR B1 · 流利观点表达 |

- 映射关系由产品提供、研发配置；Level / Unit 数量与 Unit 名称按实际取数据库，Unit 名称与 Level 描述支持多语言下发。

### 10.5 依赖

- 复用现有「报告 → 计划 → Paywall → Class」全链路与周计划组件，升级而非新建 Tab / 入口。
- 课程产能需撑住「自然 7 天解锁 3 节、Unit 约 4 个周期推进」；近期课可真实展示、地图稳步点亮。

| `word_pk_result` | 单词 PK 一局完成（假 PK，胜负按预设胜率判定） | `result=success`、`pk_outcome`（`win` / `draw` / `loss`）、`score`（己方得分）、`is_newbie_window`（`true` / `false`，是否处于新手保护期） |
| `speaking_pk_result` | 口语 PK 一局完成（假 PK，按录音完成 + 预设胜率判定） | `result=success`、`pk_outcome`（`win` / `draw` / `loss`）、`score`（己方得分）、`is_newbie_window`（`true` / `false`，是否处于新手保护期） |
| `login_result` | 登录结果（任一方式换取 Token；沿用并扩展 V1.1.0 口径） | `result`（`success` / `fail` / `cancel`）、`method`（`apple` / `google` / `phone` / `facebook` / `kakao`）、`description`（失败短码） |
