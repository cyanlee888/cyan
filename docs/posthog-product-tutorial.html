业务人员 30 分钟上手路径

1

先进入正确项目和时间范围

登录后确认当前项目、环境和日期范围。业务分析通常先看最近 7 天、30 天，再和上一个周期对比。

  • 如果数据明显异常，先确认是不是看错项目、环境或时间区间。
  • 常用入口建议收藏：Dashboards、Product Analytics、Replay、Surveys。
2

从现成看板了解业务现状

先打开团队已有 Dashboard，快速回答“整体有没有变化、变化从哪里来、影响哪些用户”。

  • 先看流量、活跃、注册/购买/核心功能使用等主指标。
  • 再按渠道、国家、设备、版本、用户类型做拆分。
3

用分析工具回答具体问题

带着问题进入 Product Analytics：趋势看变化，漏斗看转化，路径看用户怎么走，留存看用户是否回来。

4

把结论沉淀成协作材料

把常用图表保存到 Dashboard；把异常用户保存为人群或回放列表；把不确定原因转成 Survey、实验假设或需求讨论。

PostHog 的核心价值不是单个图表，而是同一份用户行为数据能在图表、回放、实验和反馈之间互相跳转，帮助业务更快形成判断。

主要功能一：分析产品使用

WWeb Analytics

适合先看网站和落地页表现，不需要复杂配置就能了解流量。

  • 看访问量、来源、页面、设备和地区。
  • 定位流量下降或某页面异常波动。
  • 从流量峰值跳到对应用户会话。

PProduct Analytics

用事件、属性和用户维度分析产品内部行为。

  • Trends：功能使用量、活跃用户、关键事件趋势。
  • Funnels：注册、下单、激活等步骤转化。
  • Paths / Lifecycle / Retention：路径、用户阶段和留存。

AActions 与 Dashboards

把自动采集的原始点击整理成业务可读指标，并保存给团队复用。

  • 用 Actions 给关键行为命名。
  • 把常用图表保存到 Dashboard。
  • 统一指标口径，减少重复提问。

主要功能二：定位体验问题

RSession Replay

像看录像一样回看用户真实操作，理解“图表背后的原因”。

  • 筛选掉队用户、异常用户、 rage click 或特定事件。
  • 结合事件时间线快速跳到关键时刻。
  • 用倍速和跳过静止时间提高观看效率。

EError Tracking

查看用户遇到的报错，判断是否影响关键转化或近期活动效果。

  • 按错误聚合、优先处理影响用户最多的问题。
  • 结合 Session Replay 复现用户当时做了什么。
  • 把高影响问题同步给对应负责人，并附上链接、时间范围和受影响用户。

LLogs 与 Activity Timeline

当你需要解释某个用户“为什么没完成操作”时，用活动轨迹补充上下文。

  • 把用户会话、事件和错误放在同一上下文里看。
  • 记录慢页面、失败接口和控制台错误等线索。
  • 追踪某个用户在问题前后的完整活动。

主要功能三：实验、开关与反馈闭环

FFeature Flags

业务侧通常不直接改开关，但需要理解哪些用户看到了哪个版本，方便分析效果。

  • 在图表或回放中按功能开关变体筛选用户。
  • 观察新功能人群和未使用人群的转化、留存、反馈差异。
  • 发现异常时，把证据同步给负责发布的同学。

XExperiments

把业务假设做成可验证的问题，围绕主指标和护栏指标判断改动是否有效。

  • 先写清楚假设：改什么、影响谁、期待提升什么。
  • 和项目负责人确认实验人群、成功指标和观察周期。
  • 结合漏斗、留存和错误率判断结果。

SSurveys

在产品内向用户提问，把定性反馈和行为数据连接起来。

  • 适合 NPS、满意度、流失原因、功能需求收集。
  • 按用户属性或行为触发问卷。
  • 查看回答者的行为、回放和相关指标。

DData Platform

当默认图表不够用时，可以使用已整理好的数据表、SQL 或 BI 看板做更深入分析。

  • 使用团队已接入的数据库、广告、SaaS 等业务数据。
  • SQL、BI、Dashboards：做更灵活的分析。
  • 把沉淀下来的人群或结果同步给运营、CRM 或客服工具。

常见任务怎么选功能

你想解决的问题 优先使用 推荐操作
最近访问量为什么下降？ Web Analytics + Trends 先按来源、页面、国家、设备拆分，再查看对应时间段的 Session Replay。
注册或购买转化卡在哪里？ Funnels + Session Replay 建立关键步骤漏斗，点击掉队人群看回放，记录卡点原因。
新功能表现是否健康？ Feature Flags + Dashboard 按开关变体拆分使用率、转化、错误率和反馈，判断是否需要继续观察或调整。
某个改版是否真的有效？ Experiments 定义主指标和护栏指标，保持实验稳定运行，避免中途随意改分流。
用户为什么不使用某功能？ Surveys + Product Analytics 筛选目标用户触发问卷，再结合用户历史行为和回放判断真实原因。
