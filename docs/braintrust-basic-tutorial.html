Braintrust 平台基础使用教程

面向新手的图文说明：从注册到追踪、Playground 评估与项目管理。内容依据官方文档整理，产品界面以线上版本为准。

官方文档：https://www.braintrust.dev/docs

1. Braintrust 是什么

示意图：应用与 Braintrust 的关系

Braintrust 是面向团队的 AI 可观测与评估平台，用于在生产环境中度量、评估并持续改进 AI 应用。典型能力包括：对比不同模型、迭代 Prompt、发现回归，以及利用真实用户数据驱动优化。

入门第一步：在官网 创建账号。

2. 组织与项目

示意图：组织与多项目结构

项目（Project）用于收纳与某一 AI 产品/场景相关的所有资源：生产日志（Logs）、实验（Experiments）、数据集（Datasets）、Prompt 与函数等。

新建项目（UI）：进入组织的项目列表 → 点击 + Project → 填写名称与可选描述 → Create。

也可通过 SDK 的 braintrust.projects.create 或 CLI bt projects create 创建；在初始化实验或 logger 时若指定新项目名，也可能自动创建项目。

在组织设置中可配置 AI 提供商密钥；项目级密钥可覆盖组织级，便于分账或隔离。

3. 推荐工作流（五步走）

示意图：Instrument → Observe → Annotate → Evaluate → Deploy

官方建议按结构化流程推进（详见 Workflow）：

  1. Instrument：为应用接入追踪，采集 LLM 调用与业务逻辑。
  2. Observe：在平台中查看日志、检索与过滤，发现模式与异常。
  3. Annotate：人工反馈、标签与修正，沉淀为高质量数据。
  4. Evaluate：用 Playground 或实验（Experiments）系统化对比改动。
  5. Deploy：上线变更并持续监控生产表现。

4. 记录第一条 Trace（链路追踪）

示意图：一条 Trace 内多类型 Span 的层级

追踪（Tracing）是观测与评估的基础：记录每次请求的输入输出、模型参数、延迟、Token 与费用、嵌套调用与错误等，数据进入 Braintrust 后可实时查看、过滤、加反馈并构建评估数据集。

接入方式概览：

  • 自动埋点：对支持的 AI 提供商与框架，尽量少改代码即可记录 LLM 调用。
  • 应用逻辑 Span：对检索、工具调用、业务步骤等自行打 Span，获得端到端视图。

支持的提供商与框架包括 OpenAI、Anthropic、Gemini、Bedrock、LangChain、LangGraph、Vercel AI SDK、DSPy 等，完整列表见 Integrations。快速上手可参考官方 Observability / Trace LLM calls 类文档。

5. Playground：交互式迭代

示意图：Playground 中的任务、评分器与数据集

Playground 是浏览器内的无代码（或低代码）实验空间：快速调整 Prompt、切换模型、挂载 Scorer（评分器）与数据集，并行跑多组结果并横向对比。

创建：进入 Playgrounds → Create empty playground 或 + Playground。

模板变量：在 Prompt 中可使用 {{input}}、{{expected}}、{{metadata.xxx}} 等（支持 Mustache / Nunjucks，详见 Write prompts 文档）。

运行：点击 Run 或使用 Cmd/Ctrl + Enter。Playground 每次运行会覆盖当前结果；若配置满意，可通过 + Experiment 提升为不可变实验快照以便长期对比。

部分高级能力（如部分标注与 Loop 优化）在 Pro / Enterprise 计划中提供，以官网说明为准。

6. 数据集与实验

示意图：数据集驱动实验与对比

数据集（Dataset）：存放带版本管理的测试用例集合，可从生产日志挑选、文件导入，或在 UI 中直接维护，供 Playground 与离线/线上评估复用。

实验（Experiment）：一次评估运行的不可变记录，便于与基线（baseline）对比分数变化、定位回归用例。也可通过 SDK、API 或 CI/CD 触发（参见 Run evaluations 文档）。

在线评分（Online scoring）：在项目自动化中按采样率对生产日志异步跑 Scorer，不阻塞主链路，结果出现在 Logs 中（详见 Score production traces）。

7. 常用 Span 类型速查

在 Trace 详情中，不同类型 Span 帮助区分计算阶段（摘自官方 Tracing 文档）：

类型含义（简述）
eval评估运行的根 Span，常包裹 task，含输入、期望输出与子 Span。
task应用逻辑单元；生产日志中根 Span 一般为 task。
llm单次大模型调用：消息、参数、Token、费用等。
function具名代码块：格式化、路由、预处理等。
tool模型发起的工具调用（API、数据库等）。
score评分器结果（线上或离线），可含 LLM-as-judge 理由。
