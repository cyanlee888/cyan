1 增长与使用结构

第二周相对第一周，会话与用户量级明显抬升；堆叠显示短会话占比高、单次访问用户仍是最大桶，增长来自各频次层一起涌入，而非明显加深粘性。

  • 关注与投放、商店更新、UXCam/埋点全量时间的对齐，避免把「采集变完整」误判为纯自然增长。
  • 长时长会话（如 >10 分钟）条带薄，需结合课程设计看是用户习惯还是中途流失。
按周：会话时长结构 — 短会话为主
按周：访问频次 — 「仅 1 次」占比最大

优化点 P1

  1. 为「首访次日回访」单独设目标与埋点校验，区分campaign用户与自然用户。
  2. 对短会话抽样录像，区分「打开即走」与「完成微任务后离开」。

2 事件趋势与集成节奏

早期区间接近全零、之后陡升，符合近期集成或放量特征。app_backgrounded、rageTap、UI Freeze、lesson_start 等同步爬升，说明挫败与卡顿信号随流量放大。

  • lesson_complete 极低而 lesson_exit 更高时，优先排查埋点定义与完成条件。
  • 跨周对比时标注「集成前无效区间」，避免误导同比。
事件计数趋势 — 与集成/放量时间强相关

优化点 P1

  1. 做一次事件字典评审（命名、触发次数、与业务语义是否一致）。
  2. 将 rageTap / UI Freeze 与版本、设备档位、网络类型交叉下钻。

3 版本表现与解读边界

2.6.1 仍占绝大多数用户；2.7.0 在部分图表中会话时长与屏数极低，而「rage/session」为 0 时更可能反映样本极少，不能直接等同于体验最优。

  • 对 2.7.0 单独监控：DAU、时长、屏数、freeze、rage，并设最小样本阈值再下结论。
  • 2.6.2 相对 2.6.1 的 rage/session 有下降趋势，可作为「已缓解点」与改动清单对照。
版本对比：会话时长与访问屏数
版本对比：rage taps / session（注意小样本版本）

优化点 P0

  1. 发布节奏表 + 版本维度日报，直到 2.7.0 渗透率与指标稳定。
  2. 若 2.7.0 存在「一屏即走」，对齐冷启动、首屏渲染与关键依赖加载。

4 屏幕重心与使用时段

AiClassRoom 相关屏在时长/会话中占比最高（约三分之一量级），与「核心教学场景」一致；周二会话占比突出，需与课表、推送、渠道核对。部分工作日柱形极低，建议排除过滤或采集缺口后再解读。

近两个月：星期 × 时段会话分布
屏幕占比 — AiClassRoom 为主力

优化点 P1

  1. 将运营日历与「周二高峰」对齐，可复制成功触达策略到其他日。
  2. 对「其他屏」聚合过大时，逐步拆分为可行动的产品模块。

5 体验雷区：冻结与愤怒点击

MainActivity 在 UI 冻结上居首；AiClassRoomActivity 在 rage tap 上为断层第一。登录相关屏在两类问题中均靠前，与「账号链路卡顿 + 挫败」叙事一致。

左：UI 冻结 by 屏（7 天）· 右：rage taps by 屏（30 天）

优化点 P0

  1. AiClassRoom：会话录像 + 元素热区，区分儿童重复点击、未响应与误触。
  2. MainActivity：主线程耗时、首包接口并行度、列表/动效掉帧专项。
  3. LoginAccount / PhoneVerify：错误提示、验证码链路、超时与重试策略。

6 转化、平台与留存

漏斗显示：登录屏 → signup_complete 约半数流失；注册完成 → lesson_start 相对跌幅最大。iOS 在「注册完成 → 开始上课」上整体优于 Android。留存上 <Day1 约一半、Day1 后骤降，且近期 cohort 多带未成熟标记，解读时注意时间窗。

按日、分平台：signup_complete → lesson_start
留存：注意 cohort 未跑满的 * 标记
漏斗：注册 → 注册完成 → 上课

优化点 P0

  1. 首课「一键到达」：注册成功页直接给明确 CTA 与进度条，减少找课成本。
  2. Android 专项：冷启动、登录 API、权限与 WebView 容器与 iOS 对齐体验。
  3. 对「0% 转化」日做样本量与发布/故障标注，避免噪声驱动决策。

7 用户旅程（Launch → Quit）

Sankey 显示启动后路径分散，且早期出现大额 Quit；LoginAccountActivity 后通往退出的流较粗，与漏斗、冻结/rage 数据三角互证。

会话流：Launch → … → Quit（关注登录后早退）

优化点 P0

  1. 为「登录后 60 秒内退出」建专项看板与告警。
  2. 减少合法入口外的循环回登录（会话恢复、token 刷新失败提示）。

8 行动优先级总览

将「主课堂体验 + 登录首启 + Android 转化」列为同一阶段主线，数据侧补齐版本切片与样本门槛。

  • P0：AiClassRoom 高 rage、MainActivity 高 freeze、登录后早退、注册→首课漏斗、2.7.0 真实用户指标。
  • P1：事件字典、短会话与「单次用户」运营策略、周二高峰复制、埋点与完成率对齐。
